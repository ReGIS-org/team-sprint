DynaSlum - Policy maker

Description:


Actors:
Researchers that want to develop slum dynamics prediction models. The model should make predictions based on given data and the reseachers should have additional data available to validate the model.

Preconditions:
- Validated model
- Policy parameters
- Visualisation settings
- Should the policy maker have knowledge of the models?

Flow:
- Load validated model
- Set policy parameters
- Run simulation
- Display predictie
- Display verandering sinds begin (groei over tijd)

Exceptions:
- Onmogelijke combinatie van beleidsparameters
- Iets veranderen aan model dat niet mogelijk is

Postconditions:
Visual analysis of prediction with ground truth on map
- Compositie (gezinnen, werkers)
- Migratie tussen slums (wat als beleid een bedrijf in slum zet)
